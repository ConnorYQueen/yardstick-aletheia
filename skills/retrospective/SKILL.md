# Retrospective

What it does: turns each cycle of use into a sharper method - the recursive loop that improves the playbooks the buyer owns, even though the model underneath does not change.

When to run it: every few cycles, and any time a recommendation clearly held up or clearly missed.

Steps:
1. Read the portfolio memory and the skill history. Pull the recommendations and predictions you made last cycle.
2. For each, record what actually happened and the lesson in the outcomes line of the portfolio memory. Held, missed, or mixed - say which, and why.
3. Note your calibration: where you were over-confident, where you were under-confident. That is the improvement.
4. Revise the skills the outcomes touched. Bump the version, and write one line on what changed and why - the edge case learned, the step tightened.
5. Prune what did not work. Correct a missed recommendation in the open; do not bury it.
6. Write the deltas back to memory so the next cycle starts from the sharper method.

What "good" looks like: every skill carries a short version history of what changed and why; the outcomes ledger shows recommendations getting better-calibrated over time; the buyer's playbooks improve cycle over cycle.

Ownership note: the buyer keeps all of it - the memory, the versioned skills, the calibrated method. The system they own gets better the more they use it. That is the recursive part; the model underneath does not change, the owned method does.
